beats.bitwrap.io · archive

Every track ever sealed to beats.bitwrap.io lives in a content-addressed share store. The archive below packages those envelopes into a single .tar.gz backup — small, self-contained, and enough to fully reconstruct the catalogue if this server is ever purged.

Archive policy
We may archive (purge from the live store) at any time and for any reason — usually because we're rolling forward with changes that would need older shares to be up-converted or re-rendered against newer versions of the software. Archived shares are not lost: they stay downloadable in the snapshot tarballs below, and visiting an archived ?cid=… URL surfaces a one-click recovery prompt that re-seals it from the latest snapshot. If you want a permanent local copy, grab a snapshot.

Restore

Replay the envelopes against any beats-bitwrap server (yours or this one):

./scripts/process-rebuild-queue.py --restore beats-snapshot-YYYYMMDD-HHMMSS.tgz

That's it — every track shows up at ?cid=… again, and audio re-renders the first time a listener opens it.
